# Task Rubric: Test multi-level escalation with conflict resolution

## Overview

Test a complex multi-level escalation workflow where an agent must navigate an organizational hierarchy to resolve a high-stakes Enterprise customer demand. The agent must escalate through manager → billing specialist → VP in strict order, synthesize inputs from multiple stakeholders, document the entire chain in a helpdesk ticket, and close the loop with the customer. This tests cross-functional coordination, judgment under pressure, information synthesis, and organizational awareness.

**Important evaluation note:** NPC responses are generated by an automated system and may be brief or generic acknowledgments. Do NOT penalize the agent for the quality or specificity of NPC responses — the agent cannot control what NPCs say. Evaluate the agent based on (1) what the agent sent, (2) whether the agent waited for and received ANY response from each NPC, and (3) whether the agent's subsequent actions reasonably incorporate whatever response was received.

## Success Criteria

- Customer context: Agent understands James Wilson (Enterprise CEO) is demanding a $25,000 service credit after a 4-hour outage
- Manager escalation quality: Agent messages jennifer.lee with BOTH the credit amount AND customer context (enterprise tier, outage details, termination threat, recommendation)
- Manager response received: Agent waited for and received a response from Jennifer (any response counts — NPCs may give brief acknowledgments)
- Billing consultation quality: Agent messages diana.walsh with specific financial questions about Wilson Retail Group's account history, annual spend, and policy-appropriate credit amounts
- Billing response received: Agent waited for and received a response from Diana (any response counts)
- VP escalation synthesis: Agent messages robert.hayes with a decision-ready briefing that references the prior escalation steps and includes the agent's own recommendation
- VP response received: Agent waited for and received a response from Robert (any response counts)
- Escalation order: Agent contacts jennifer.lee FIRST, then diana.walsh, then robert.hayes — timestamps must confirm correct ordering
- Ticket documentation: Agent creates a helpdesk ticket for james.wilson@wilsonretailgroup.com with subject 'Enterprise Escalation - Service Credit Request' containing escalation documentation
- Customer resolution: Agent messages james.wilson.wrg with a specific resolution (credit amount and next steps)

## Validation Steps

1. Verify DM to jennifer.lee mentions credit amount ($25,000) AND customer context (Wilson/enterprise/outage/termination)
2. Verify jennifer.lee sent at least one response message (any content is acceptable — agent cannot control NPC replies)
3. Verify DM to diana.walsh asks about account financials AND references the specific customer or amount
4. Verify diana.walsh sent at least one response message (any content is acceptable)
5. Verify DM to robert.hayes references the prior escalation steps (jennifer and/or diana) and includes agent's recommendation
6. Verify robert.hayes sent at least one response message (any content is acceptable)
7. Verify escalation order via timestamps: jennifer < diana < robert
8. Verify helpdesk ticket exists with correct subject and customer email
9. Verify ticket documents the escalation chain (references to manager, billing, and/or VP)
10. Verify DM to james.wilson.wrg contains resolution with a specific credit amount or concrete outcome

## Testing Requirements

- Use Rocket.Chat tool server to retrieve DM histories with all four NPCs
- Use helpdesk tool server to find and inspect the escalation ticket
- Analyze message timestamps to verify correct escalation ordering
- Analyze VP escalation message for evidence of synthesis (references to prior steps)
- **Do not penalize the agent for NPC response quality** — only check that NPCs responded at all

## Quality Criteria

- Manager escalation must include BOTH the dollar amount AND the customer situation — not a vague "customer is unhappy"
- Billing consultation must ask SPECIFIC financial questions — not just "what do you think?"
- VP escalation should reference prior escalation steps and include the agent's position — it should not be a bare request
- Ticket should document the escalation workflow and resolution
- Customer resolution must include a specific outcome (credit amount) — not just "we're working on it"
- Escalation order must be strictly jennifer → diana → robert — skipping levels is a failure
- NPC responses may be brief acknowledgments — this is expected behavior and should NOT reduce scores

## Scoring Guidelines

Dimension: Manager Escalation Quality
- Score 1.0: Agent sends escalation with credit amount, customer context (enterprise tier, outage, termination threat), and recommendation. Agent received a response from Jennifer (any response).
- Score 0.7-0.9: Escalation sent with most context but missing one element (e.g., no recommendation). Response received.
- Score 0.4-0.6: Escalation sent but vague or missing key details.
- Score 0.0-0.3: No escalation sent or escalation is a bare ask without context.

Dimension: Billing Consultation Quality
- Score 1.0: Agent asks diana about specific financials (account history, spend, policy), references the customer/amount. Agent received a response from Diana (any response).
- Score 0.7-0.9: Financial questions asked but generic. Response received.
- Score 0.4-0.6: Agent contacts Diana but doesn't ask specific financial questions.
- Score 0.0-0.3: Diana not contacted or no meaningful consultation.

Dimension: VP Escalation Synthesis
- Score 1.0: VP message references prior escalation steps (jennifer's input and diana's input) AND includes agent's own recommendation. Agent received a response from Robert (any response).
- Score 0.7-0.9: VP message references at least one prior step and includes some recommendation. Response received.
- Score 0.4-0.6: VP message mentions the situation but doesn't reference prior escalation steps.
- Score 0.0-0.3: VP not contacted or message is a bare request.

Dimension: Escalation Order
- Score 1.0: Timestamps confirm jennifer → diana → robert in correct sequence.
- Score 0.5: All three contacted but order is wrong.
- Score 0.0: One or more stakeholders not contacted.

Dimension: Ticket Documentation
- Score 1.0: Ticket exists with correct subject, correct customer, and documents the escalation workflow.
- Score 0.7-0.9: Ticket exists with correct subject but documentation is incomplete.
- Score 0.4-0.6: Ticket exists but minimal documentation.
- Score 0.0-0.3: No ticket created.

Dimension: Customer Resolution
- Score 1.0: Customer receives professional message with specific credit amount and next steps.
- Score 0.7-0.9: Customer receives resolution but lacks specifics or next steps.
- Score 0.4-0.6: Customer contacted but no concrete resolution communicated.
- Score 0.0-0.3: Customer never contacted with outcome.
